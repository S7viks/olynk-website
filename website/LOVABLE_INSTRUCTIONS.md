## What is Lovable?

**Lovable.dev** is an AI-powered tool that generates complete web applications from a single text prompt. It's perfect for rapid prototyping, testing designs, and getting a working website in minutes.

**Tech Stack:** React 18, TypeScript, Tailwind CSS, Framer Motion, Vite

## Step-by-Step Instructions

### 1. Go to Lovable.dev
Visit [https://lovable.dev](https://lovable.dev) and sign in.

### 2. Open the Prompt File
Open the file **`LOVABLE_PROMPT.md`** in this folder.

### 3. Copy Everything
- Select all content in `LOVABLE_PROMPT.md`
- Copy to clipboard (Ctrl+A, Ctrl+C on Windows; Cmd+A, Cmd+C on Mac)

### 4. Paste into Lovable
- In Lovable, create a new project
- Paste the entire prompt into the main input field
- Click "Generate" or "Build"

### 5. Wait for Generation
Lovable will:
- Set up the project structure
- Create all components
- Implement the design system
- Add all 15 homepage sections
- Configure routing and navigation
- Set up forms and interactions

**Typical generation time:** 2-5 minutes

### 6. Preview & Iterate
- Once generated, you'll see a live preview
- Test the website on different screen sizes
- Make refinements by chatting with Lovable
- Export the code when satisfied

## Common Issues & Solutions

### Issue: "Generation failed"
**Solution:** Try again. Sometimes Lovable has temporary issues. If it persists, break the prompt into smaller chunks and ask Lovable to "continue building" each section.

### Issue: "Some sections are missing"
**Solution:** Chat with Lovable: "You missed Section X. Please implement it as specified in the original prompt."

### Issue: "Colors don't match"
**Solution:** Chat with Lovable: "Update all colors to match the design system: Primary Blue #2B5288, Dark Navy #223148, etc."

### Issue: "Mobile version looks broken"
**Solution:** Chat with Lovable: "Fix mobile responsiveness. Stack all grid sections vertically on mobile, reduce font sizes, and ensure touch targets are 44x44px minimum."

## Alternative: Manual Development

If Lovable doesn't meet your needs, use **`BUILD_PROMPT.md`** instead and build manually with Next.js.

**When to use manual development:**
- Need custom backend integrations
- Require specific CMS (Contentful, Sanity)
- Want more control over architecture
- Building for production with high traffic
